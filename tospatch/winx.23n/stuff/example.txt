Beispiel einer Umgebung in der WINX eingesetzt wird
---------------------------------------------------

Voraussetzung: Standard TOS (kein MultiTOS, Mag!C, GENEVA, ...)

AUTO\				; = Bootprogramme =
	GEMRAM.PRG		; Stellt GEM im RAM zum Patchen bereit
	GEMRAM.ENV		; Textdatei mit Environmentvariablen fuers GEM
	WINX.PRG		; GEM-Patches
	WINX.INF		;    Optionen für GEM-Patches
	ICFS.PRG		; Iconify-Server
	DPATTERN.PRG	; Schreibtischhintergrund
	DPATTERN.RSC	;    Auswahl von Schreibtischhintergründen
	GDOS.PRG		; Beliebiges GDOS zur Einbindung von Fonts

CPX\				; = Konfigurationsmodule =
	WINX.CPX		; WINX Eigenschaften
	ICFS.CPX		; Iconify
	DPATTERN.CPX	; Schreibtischhintergrund
	WINCOLOR.CPX	; Fensterfarben und -effekte
	IDATE.CPX		; Datumsformat
	ITIME.CPX		; Zeitformat
	INUMBER.CPX		; Zahlenformat

GUIDES\				; = Dokumentation =
	WINX.HYP		; WINX
	WINX.REF		;
	ICFS.HYP		; Iconify-Server
	ICFS.REF		;
	DPATTERN.HYP	; Schreibtischhintergrund
	DPATTERN.REF	;
	
GEMSYS\				; = GDOS-Erweiterungen =
	SRCP09.FNT		; verschiedene Bitmapfonts
	SRCR09.FNT
	SRCR10.FNT
	WDPR08.FNT
	WDPR10.FNT
	
ASSIGN.SYS			; GDOS-Infos (z.B. Fonts)
ST-GUIDE.ACC		; Viewer fuer Dokumentation
XCONTROL.ACC		; Modulares Kontrollfeld


Mögliche Modifikationen:

Statt GEMRAM können auch andere Programme benutzt werden um das GEM im 
RAM zu installieren (z.B. ROMRAM, VRAM, ROMSPEED, OUTSIDE). Bis ein-
schlieŞlich der TOS-Version 1.04 kann man im Prinzip auch ganz auf 
GEMRAM verzichten.

ICFS braucht man nur, wenn man die Möglichkeit haben will seine Fenster 
zu ikonifizieren (auf IcongröŞe schrumpfen zu können).

DPATTERN wird nur benötigt, wenn man einen eigenen Schreibtischhinter-
grund einstellen will.

Ein GDOS wird zum Einstellen eines Fensterrahmenfonts benötigt. Benutzt 
man einen Bitmapfont, ist aber auch das nicht notwendig. Zum GDOS ge-
hört aus seine Konfigurationsdatei ASSIGN.SYS.

XCONTROL ist zum Einsatz von WINX und DPATTERN nicht zwingend er-
forderlich, d.h. nachdem man die Programme über ihre CPXe konfiguriert 
hat, kann man XCONTROL abschalten, wenn man den Speicher benötigt. 
Will man WINCOLOR.CPX und die IFormat-CPXe benutzen, dann ist XCONTROL 
jedoch notwendig.

ST-Guide ist nur notwendig, wenn man die Hypertextdokumente im Guides- 
Ordner lesen will.
